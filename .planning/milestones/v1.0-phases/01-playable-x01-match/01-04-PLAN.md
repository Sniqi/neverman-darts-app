<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| profile name (user text) → IndexedDB + DOM | Player-typed names are persisted and rendered in setup/bull-off lists |
| setup config (user choices) → reducer | Mode/out/legs/sets choices configure the state machine |
| Dexie open → app | IndexedDB may be unavailable (private mode/quota) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Spoofing/XSS | Profile/guest name rendered in PlayerPicker, ProfileManager, BullOffOrder | mitigate | All names rendered via Svelte `{...}` interpolation (auto-escaped); no `{@html}`, no `innerHTML` (RESEARCH Security Domain: XSS via player name). |
| T-04-02 | Denial of Service | Dexie open failure during profile CRUD (private browsing/quota) | mitigate | `src/db/profiles.ts` wraps DB access in try/catch; on failure the setup screen still works with guests only and surfaces a non-blocking inline error (RESEARCH V7 Error Handling). |
| T-04-03 | Tampering | Starting a match with 0 players / invalid config | mitigate | "Spiel starten" disabled until ≥1 player; config values constrained to the allowed sets (301/401/501, single/double, legs 1–9). |
| T-04-04 | Information Disclosure | Guest data accidentally persisted | mitigate | Guests carry `isGuest: true` and the guest add path performs no `db.profiles` write (PROF-02); acceptance criterion greps for the absence of a DB write in that path. |
| T-04-SC | Tampering | npm installs (supply chain) | accept | No new packages introduced (hand-rolled bull-off sort, no SortableJS) — all deps already vetted in RESEARCH Package Legitimacy Audit (all Approved). |
</threat_model>
