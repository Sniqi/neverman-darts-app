<objective>
Close the engine-layer correctness gaps that break undo reliability (ENG-05) and numpad
validation (INP-02). Two independent defects live in pure, TDD-covered engine modules:

1. CR-07 + WR-01 — event-log semantics in `reducer.ts`:
   - `START_MATCH` appends to the existing event log instead of starting fresh, so UNDO at
     the start of match #2 replays match #1 and resurfaces the previous match-complete state.
   - `CONFIRM_VISIT` is a no-op that is still appended to the log, so the first UNDO press
     after any confirmed visit silently removes only that no-op, requiring two presses to
     undo one dart.
2. CR-02 — `IMPOSSIBLE_3DART` in `impossible-scores.ts` is missing 163 and 166, so the
   numpad currently accepts those physically impossible 3-dart totals and corrupts state.

Purpose: Restore the single-mental-model undo (D-06, ENG-05) and complete the impossible-score
guard (INP-02). These are pure-logic fixes verifiable entirely by Vitest unit tests.
Output: Corrected `reducer.ts` and `impossible-scores.ts` plus extended unit tests that lock
the corrected behavior.
</objective>
