<objective>
Close CR-08: `ProfileManager.svelte` is fully implemented and unit-tested but is not mounted in
any route or component (grep finds it only in its own test file). `PlayerPicker` reads persistent
players from `profilesLive()` — a Dexie table that starts empty and can never be populated because
there is no UI path to create a profile. The only usable player type today is a guest, so PROF-01
(create/edit/delete persistent profiles) is unreachable in the running app and FLOW-01 (add named
or guest players) is only half satisfied.

The fix mounts `ProfileManager` inside the setup flow. `ProfileManager` and `PlayerPicker` already
read/write the same `db.profiles` table via `profilesLive()`, so a profile created in ProfileManager
will appear live in PlayerPicker's profile dropdown with no extra wiring.

Purpose: Make named-player creation reachable from the UI (PROF-01) and complete the setup flow
(FLOW-01).
Output: `MatchSetup.svelte` with a reachable profile-management section rendering `ProfileManager`.
</objective>
