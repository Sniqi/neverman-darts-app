<objective>
Repair the correction window (INP-04) so the visit flow actually progresses. Two compounding defects make it a permanent blocker:

- CR-04: the `$effect` calls `startTimer()`, which reads the `$state` variable `elapsed`, making `elapsed` a tracked dependency. Svelte re-runs the effect on every requestAnimationFrame tick (~60fps); the cleanup calls `stopTimer()` (cancelling the pending 2500ms `setTimeout`) before it can fire, then a fresh timer starts. Auto-dismiss never fires.
- CR-03: pressing 'Korrigieren' sets `paused = true` and renders only a hint overlay. The overlay (`position: absolute; inset: 0; z-index: 10`) fills `.panel-relative`, which contains the 'Rückgängig' (undo) button it tells the user to press — so the undo button is unclickable. While paused `handleOutsideClick` is a no-op and no dismiss/resume control is rendered. The match UI is permanently inaccessible after the first 'Korrigieren' press.

The fix: wrap `startTimer()` in `untrack(() => startTimer())` so the effect runs once per visibility change; render an explicit 'Fertig' control while paused that calls `confirm()`; and make `handleOutsideClick` dismiss the window regardless of paused state. Update the component test to prove auto-dismiss with effects flushed (the current fake-timer test is a false green — IN-08) and to cover the paused-escape control.

Purpose: Make the visit auto-finalize and the correction window escapable (INP-04 / D-05).
Output: A correction window that auto-dismisses once and is always escapable.
</objective>
