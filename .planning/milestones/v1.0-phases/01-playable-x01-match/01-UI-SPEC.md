# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Playable X01 Match.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (SvelteKit — shadcn is React-only) |
| Preset | not applicable |
| Component library | none — hand-built Svelte components |
| Icon library | none in Phase 1 — use Unicode symbols or inline SVG where needed |
| Font | system-ui stack: `system-ui, -apple-system, sans-serif` |

**Rationale:** The stack is SvelteKit + Svelte 5. shadcn targets React/Radix. No third-party component library is introduced in Phase 1. CSS is authored as Svelte scoped styles + a minimal global token sheet.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, visit strip dart separators, label-to-value gaps |
| sm | 8px | Compact element spacing: button internal padding, chip gaps |
| md | 16px | Default element spacing: card padding, form field gaps |
| lg | 24px | Section padding: score panel internal padding, setup section gaps |
| xl | 32px | Layout gaps: portrait column spacing, setup screen section breaks |
| 2xl | 48px | Major section breaks: between score panel and dartboard in portrait |
| 3xl | 64px | Page-level spacing: not used in Phase 1 |

**Exceptions:**

| Element | Value | Reason |
|---------|-------|--------|
| Dartboard touch target minimum | 44px | WCAG 2.5.5 minimum touch target; double/triple ring SVG hit areas must each span at least 44px at rendered board size |
| Undo button touch target | 48px minimum height | Operated mid-match under pressure; larger target reduces mis-taps |
| Number keypad digit keys | 64×64px minimum | Comfortable fat-finger digit entry on a 10" tablet |
| Visit strip dart slot | 48px height | Tap-to-correct requires a reliably hittable target |
| Bull-off player order slots | 56px height | Drag-to-reorder list items on tablet |

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px | 400 | 1.5 | Setup labels, form fields, player names in setup, general UI text |
| Label | 14px | 400 | 1.4 | Visit strip dart entries (T20 · 20 · –), checkout suggestion line, numpad input field, correction-window timer |
| Heading | 20px | 600 | 1.2 | Section headings in setup screen, current-player name in score panel |
| Display | 48px | 600 | 1.0 | Remaining score for the active player (the primary at-a-glance number during play) |

**Notes:**
- Only 4 sizes and 2 weights are declared. Do not introduce additional sizes or weights in Phase 1.
- The 48px display size is intentionally large: the remaining score is the most-read number during a match.
- Segment number labels on the dartboard SVG (1–20) use 14px / 400 / `pointer-events: none` — decorative only, taps fall through to scoring paths.

## Color

Dark mode is the only mode in Phase 1. PLAT-04 locks "native dark mode design."

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#111318` | Page background, dartboard background ring, screen body |
| Secondary (30%) | `#1e2027` | Score panel card background, setup form card background, numpad key background, player profile list background |
| Accent (10%) | `#e8a020` | Reserved list — see below |
| Destructive | `#c0392b` | Bust indicator only: visit strip turns red on bust; "Daneben (Bust)" confirmation text |

**Accent `#e8a020` is reserved exclusively for:**
1. Active player highlight border on the score panel card
2. Checkout suggestion text (the route string, e.g. "T20 T20 Bull")
3. Bull — inner bull SVG fill (25/50 target, the highest-value zone)
4. Numpad "Bestätigen" (Confirm) button background
5. Undo button icon/stroke color

**Accent is NOT used on:** general buttons, all dartboard segments, player names, headings, or any decorative element.

**Surface colors in the dartboard SVG:**

| Segment zone | Color | Notes |
|---|---|---|
| Single scoring area (inner + outer) | `#2d2d2d` | Neutral dark, matches board feel |
| Triple ring | `#1a5c2e` (green) | Standard board coloring, high-contrast against dark background |
| Double ring | `#8b1a1a` (red) | Standard board coloring, high-contrast against dark background |
| Wire/dividers | `#444444` | Thin decorative stroke, `pointer-events: none` |
| Inner bull | `#e8a020` | Accent — highest value target, visually distinct |
| Outer bull | `#1a5c2e` | Matches triple ring green |
| Active (just-tapped) segment flash | `rgba(255,255,255,0.35)` overlay for 300ms | Brief white overlay for tap feedback; cleared automatically |
| Miss zone (outside double ring) | transparent hit overlay | No visible color change on miss tap |

**Text on all surfaces:** `#f0f0f0` (near-white). Do not use pure `#ffffff` — the slight warmth reduces eye strain under dark mode.

## Layout Contracts

Phase 1 has three views. Each has a fixed layout contract.

### View 1: Match Setup (`/` or `/setup`)

Single scrollable column. Max width 480px, centered on wider screens.

**Sections (top to bottom):**
1. App title bar: "Neverman Darts" — 20px heading, no back button
2. Player picker: tap to add profile players or add guest; max 4 entries; reorderable via drag (bull-off order is set after setup, not here)
3. Game mode chips: "301" / "401" / "501" — 3 horizontally tiled full-width chips
4. Out-rule toggle: "Single Out" / "Double Out" — segmented control, 2 options
5. Format stepper: "Legs" stepper (1–9) + optional "Sets" toggle; when sets enabled, a second stepper appears
6. "Spiel starten" button — full width, accent background

**No modal dialogs on the setup screen.** All choices are inline.

### View 2: Scoring View (`/match`)

Two layouts, chosen by `orientation: landscape` media query.

**Portrait layout (phones / tablet in portrait):**
```
┌──────────────────────────────┐
│  Score Panel (current + all) │  ← ~35vh
├──────────────────────────────┤
│  Visit Strip + Checkout Line │  ← ~8vh
├──────────────────────────────┤
│  Dartboard (SVG, square)     │  ← fills remaining height
│  [Numpad toggle at bottom]   │
└──────────────────────────────┘
```

**Landscape layout (tablet in landscape, primary play orientation):**
```
┌──────────────────────┬───────────────────┐
│  Score Panel         │  Dartboard (SVG)  │
│  + Visit Strip       │                   │
│  + Checkout Line     │  [Numpad toggle]  │
└──────────────────────┴───────────────────┘
```
Score panel takes 38% width; dartboard takes 62% width.

**Score panel card — per player:**
- Player name: 20px / 600
- Remaining score: 48px / 600 (display size, active player) or 20px / 600 (inactive players — heading size provides sufficient contrast without adding a 5th font size)
- Legs won this set: shown as dot indicators or "L: 2"
- Sets won: shown if sets mode enabled
- Checkout suggestion: 14px / 400 / accent color, one line only, e.g. "T20 T20 Bull"

Active player card has a 2px accent-colored left border.

**Visit strip** (below score panel, above dartboard):
- Shows the 3 dart slots of the current visit: "T20 · 20 · –"
- Each slot is 48px tall, tappable to undo that specific dart (undo the last dart only — tapping an earlier slot undoes back to that dart)
- Current dart slot: empty dash "–"
- Font: 14px / 400

**Numpad toggle:**
- A toggle icon/label at the bottom of the dartboard area switches between the SVG board and a numeric keypad
- App remembers the last-used mode per player within the match (Claude's discretion: store in match store state, not persisted to DB in Phase 1)

**Numpad layout (when toggled):**
- Standard phone numpad: 1–9, 0, plus "C" (clear) and "✓" (confirm)
- Digit keys: 64×64px minimum
- Confirm button: full width of bottom row, accent background, label "Bestätigen"
- Invalid total (impossible scores: 179, 178, 176, 175, 173, 172, 169 in the 3-dart impossible list, or > 180): input field shakes / turns red, does not submit

### View 3: Bull-Off Order Screen (`/bulloff`)

Shown between setup and the first match. Displays a sorted list of player cards. User taps players in the order they threw closest to the bull (first = first to throw). Or drags to reorder.

- Player cards: 56px height, player name 16px / 400
- "Spielreihenfolge bestätigen" button: full width, accent background

### Correction Window (overlay layer)

After a visit ends, a semi-transparent overlay appears over the score panel (not the board) for 2.5 seconds showing the just-completed visit:
- Background: `rgba(30, 32, 39, 0.92)` (secondary with high opacity)
- Shows the visit darts and total: "T20 · T20 · T20 → 180"
- "Korrigieren" link in 14px opens the visit for editing (keeps overlay open)
- Tapping outside or waiting 2.5 s passes the turn
- Timer display: a thin progress bar at the top of the overlay, draining in 2.5 s

### Darts-at-Double Dialog (numpad finishing visit only)

When a numpad-entered visit wins a leg, a bottom sheet slides up:
- "Wie viele Darts auf die Doppel?" — 16px body
- Three large tap targets: "1 Dart", "2 Darts", "3 Darts" (each 56px height)
- "Doppelversuch abgebrochen" option for visits that didn't finish (auto-skipped for non-finishing numpad visits)

## Dartboard Visual Spec

SVG dartboard dimensions: square `viewBox="0 0 400 400"`, center `(200, 200)`, outer double ring edge at radius 195px (leaves 5px margin).

| Ring | Inner radius | Outer radius | Visual treatment |
|------|---|---|---|
| Inner bull (50) | 0 | 14.4px | Accent fill `#e8a020` |
| Outer bull (25) | 14.4px | 36.5px | Green `#1a5c2e` |
| Inner single | 36.5px | 186px (varies) | Alternating dark grey, `#2d2d2d` |
| Triple ring | 186px | 209px | Green/red alternating per segment standard |
| Outer single | 209px | 303px | Alternating dark grey |
| Double ring | 303px | 325px | Red/green alternating per segment standard |
| Miss zone | 325px | 390px (SVG edge) | Transparent, but pointer-events: all — tapping here = 0 |

**Enlarged ring rationale (D-01):** Triple ring is drawn at ~2.3× real proportional width; double ring at ~2.1×. These are intentional design choices for touch accuracy — not a realistic board.

**Segment number labels:** positioned at radius ~345px from center, 14px / 400, `pointer-events: none`, fill `#888888`.

**Hit feedback:** On `pointerdown`, the tapped segment receives a white overlay fill (`rgba(255,255,255,0.35)`) for exactly 300ms via CSS animation, then reverts. No sound in Phase 1.

## Interaction Contracts

| Interaction | Behavior |
|---|---|
| Tap dartboard segment | Instant score: no confirm step (D-03). Segment flashes 300ms. Dart added to visit strip. Visit auto-finalizes after 3rd dart, bust, or leg win → correction window appears. |
| Tap outside double ring (miss zone) | Scores 0, labelled "Daneben" in visit strip. Same flow as any dart. |
| Undo (visit strip tap or dedicated undo button) | Removes the last dart from the current visit. If visit strip is empty (no darts in this visit), undo steps back to the previous visit. Unlimited depth. Leg/set counts are correctly reverted. |
| Numpad toggle | Flips between dartboard SVG and numeric keypad. Animated slide transition (200ms ease-out). Last-used mode per player remembered in match state. |
| Numpad confirm | Validates total (must be 0–180, must be achievable in 3 darts). Impossible totals: reject with shake animation, no state change. Valid total: committed as a visit. |
| Correction window countdown | 2.5s progress bar. Tap "Korrigieren" to pause and edit. Tap outside to confirm early and pass turn. |
| Setup "Spiel starten" | Disabled until at least 1 player added. Navigates to bull-off screen. |
| Bull-off order confirm | Navigates to scoring view. First player in list throws first. |
| Match win | Score reaches 0 with valid checkout → full-screen win overlay showing player name + match summary. "Neues Spiel" button and "Zur Übersicht" (for later phases). |
| Bust | Visit strip row turns destructive red. Correction window shows "Überworfen!" label. After 2.5 s, turn passes without score change. |

## Animation & Motion

| Element | Animation | Duration | Easing |
|---|---|---|---|
| Segment tap feedback | White overlay fade-in + fade-out | 300ms total | ease-out |
| Correction window appear | Fade-in + slide up 8px | 200ms | ease-out |
| Correction window dismiss | Fade-out | 150ms | ease-in |
| Numpad toggle | Slide in from bottom / slide out to bottom | 200ms | ease-out |
| Darts-at-double bottom sheet | Slide up from bottom | 250ms | ease-out |
| Bust indicator | Visit strip row background fades to destructive red | 300ms | ease-out |
| Numpad invalid shake | 3-cycle horizontal shake on input field | 400ms | ease-in-out |
| Match win overlay | Fade-in | 300ms | ease-out |

**Principle:** All animations are sub-400ms. No looping animations during active play (they distract). No animations that block user input.

## Accessibility Contracts

| Requirement | Implementation |
|---|---|
| Touch targets | All interactive elements: minimum 44×44px. Double/triple ring hit areas: backed by enlarged invisible hit overlays if SVG path area < 44px at rendered size. |
| Contrast | Body text `#f0f0f0` on `#111318`: 16.2:1. Score display `#f0f0f0` on `#1e2027`: 14.8:1. Both exceed WCAG AA (4.5:1). Accent `#e8a020` on `#1e2027`: 8.4:1 — passes for both AA and AAA. Destructive `#c0392b` on `#111318`: 5.2:1 — passes AA. |
| Keyboard navigation | Setup form and profile CRUD must be keyboard-navigable (Tab order + Enter/Space). Scoring view: keyboard navigation is a nice-to-have, not required — it is a touch-first app. |
| Screen wake lock | INP-05: `navigator.wakeLock.request('screen')` acquired on match start; re-acquired on `visibilitychange`. Silently no-ops if unsupported. |
| `pointer-events: none` | Applied to all SVG decorative elements (number labels, wire dividers) so taps fall through to scoring paths. |
| `touch-action: none` | Applied to the dartboard SVG element to prevent scroll/zoom from intercepting `pointerdown`. |
