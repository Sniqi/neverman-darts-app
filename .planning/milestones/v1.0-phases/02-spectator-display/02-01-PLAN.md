<objective>
Lay the data foundation and the failing-test baseline for the spectator display. This is the Wave 0 (tests-first) plan combined with the engine-side data the first real vertical slice needs: leg/match average computation and the `legStartVisitIndex` that lets the current leg's visits be isolated from prior legs.

Purpose: The spectator panels (DISP-03) must show "Ø Leg" and "Ø Match" per player. The reducer does NOT reset `player.visits` at leg start (verified in reducer.ts lines 309-325) — so leg average requires knowing where the current leg began. This plan adds that one field and the pure averaging functions, plus the `DisplayStore` subscriber that the `/display` route will mount, and writes the failing e2e baseline that the whole phase drives toward (DISP-05).

Output: `src/engine/averages.ts`, an extended `MatchState` with `legStartVisitIndex`, `src/stores/display.svelte.ts`, and three test files (two unit, one failing e2e).
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| scoring tab → spectator tab (BroadcastChannel) | Same-origin only; another browser context receives MatchState snapshots |
| localStorage → DisplayStore (JSON.parse) | Untrusted-shaped string deserialized into MatchState on hydrate |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | `JSON.parse` of localStorage snapshot in DisplayStore.connect() | mitigate | Parse inside try/catch; on failure fall back to `state = null` (idle screen). No deep schema validation required (ASVS L1, same-origin local app) per RESEARCH Security Domain. |
| T-02-02 | Spoofing | BroadcastChannel('neverman-match') message source | accept | BroadcastChannel is same-origin only; no cross-origin injection possible. ASVS L1 requires no further control. |
| T-02-03 | Denial of Service | localStorage quota / private-mode write failure | accept | Snapshot write (Plan 02 publisher) wrapped in try/catch; live channel remains authoritative; cold-start simply shows idle. Non-fatal by design. |
</threat_model>
