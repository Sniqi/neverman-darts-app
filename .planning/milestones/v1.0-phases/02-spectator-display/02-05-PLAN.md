<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| scoring window → spectator window | State crosses via `BroadcastChannel('neverman-match')` and `localStorage('neverman-match-snapshot')`. Both are **same-origin only** by browser spec — no cross-origin sender can publish to this channel or write this key. |
| opened second window (`/display`) → opener | `window.open` returns a handle whose `opener` could enable reverse-tabnabbing if the target were untrusted. The target is **same-origin app content** (`${base}/display`). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | `display.svelte.ts` snapshot/channel payload | accept (existing; out of scope here) | Payload shape-validation (WR-08) is a separate, lower-severity warning not in this gap plan's scope. Same-origin-only channel means no untrusted producer; the only producer is this same app. No high-severity exposure. |
| T-02-06 | Elevation of Privilege | `SpectatorChooser.openSecondWindow()` | mitigate | Reverse-tabnabbing guard preserved by nulling `win.opener` after open (Task 3). Dropping `noopener` from the features string is offset by the manual `opener = null`; target is same-origin so no cross-origin opener leak is possible. |
| T-02-INFO | Information Disclosure | BroadcastChannel('neverman-match') | accept | Channel is same-origin; data is local match scores with no PII or secrets. `$state.snapshot()` produces a plain serializable clone of already-local data — no new exposure (Task 1). |
| T-02-SC | Tampering | npm/pip/cargo installs | n/a | No package installs in this plan; all changes are edits to existing source/test files. No supply-chain surface introduced. |

**No high-severity threats.** Evidence: (1) every data path is same-origin only (BroadcastChannel and localStorage are origin-scoped by the HTML spec); (2) the payload is local, non-PII match state generated by this same app; (3) `$state.snapshot()` is a plain structured-clone-safe copy, not a new sink; (4) the only privilege boundary (opened window opener) is explicitly nulled and points at same-origin content; (5) no new dependencies are added. `security_block_on: high` finds nothing to block.
</threat_model>
