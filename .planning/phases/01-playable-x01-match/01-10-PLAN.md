<objective>
Fix the inner-bull scoring domain bug (CR-01). `classifyHit` currently returns `{ multiplier: 2, segment: 50 }` for the inner bull; because every scoring site computes points as `multiplier * segment`, a bull tap scores 100 instead of 50. This makes a legitimate double-out finish on Bull (from remaining=50) register as an overshoot bust, and makes every checkout route ending in 'Bull' (170, 167, 164, 161, 50) produce a bust when followed.

This plan changes the canonical inner-bull encoding to `{ multiplier: 2, segment: 25 }` (double-bull = 2×25 = 50, and `multiplier === 2` already satisfies the double-out finish check in bust.ts), updates the two tests that locked in the wrong rule, removes the now-dead `segment === 50` branch in bust.ts, and updates the VisitStrip `formatDart` label so the new encoding still renders 'Bull'.

Purpose: Restore correct X01 scoring so bust handling (ENG-04) and checkout suggestions (ENG-07) work on Bull finishes.
Output: Corrected engine encoding + tests asserting the correct rule + correct display label.
</objective>
