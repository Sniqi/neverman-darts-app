<objective>
Deliver the thinnest viable end-to-end vertical slice of the spectator display: a dart entered on `/match` appears live on an open `/display` window, rendered as a TV-broadcast grid of player panels. After this plan a real user can open the spectator route in a second browser tab and watch scores update on every dart, with correct layouts for 1–4 players.

Purpose: This is the MVP happy path for the whole phase. It wires the locked sync primitive (BroadcastChannel + localStorage snapshot from CLAUDE.md) into `MatchStore.dispatch()`, then builds the minimal render target: `/display` route → `MatchHeader` (D-03) + a grid of `PlayerPanel`s (D-01 equal-split, D-04 per-panel content, D-02 active highlight) → idle fallback (D-11). It covers DISP-03 (scores, legs/sets, names, active player, averages), DISP-04 (TV typography + 1–4 layouts), and the live half of DISP-05.

Output: BroadcastChannel publisher in `match.svelte.ts`; new `/display` route; `PlayerPanel`, `MatchHeader`, `IdleScreen` components; browser component tests for the panel.
</objective>
