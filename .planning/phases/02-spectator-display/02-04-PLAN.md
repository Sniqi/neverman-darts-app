## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| scoring view → new browser window (window.open) | A second same-origin window is opened to /display |
| user gesture → Fullscreen API | requestFullscreen must originate in a click/tap handler |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-06 | Spoofing (reverse tabnabbing) | `window.open('${base}/display', '_blank', 'noopener,noreferrer')` | mitigate | Always pass `'noopener,noreferrer'`; `window.opener` is prohibited (CLAUDE.md). Acceptance greps for the noopener flag. |
| T-02-07 | Denial of Service / UX | Fullscreen request denied or popup blocked | mitigate | requestFullscreen wrapped in try/catch — display works in a normal window on denial; window.open null → inline "Bitte Popups…" message, no crash. |
| T-02-04 | Injection (XSS) | Any user-name in chooser-adjacent display copy | mitigate | Interpolation only; `{@html}` prohibited (T-03-04); grep gate enforced. |
</threat_model>
