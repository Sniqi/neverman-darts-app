## Typography

### Spectator Display (`/display` route)

Typography is tuned for legibility on a 27" monitor at 3 m (DISP-04 requirement). All sizes use `clamp()` for fluid scaling across window sizes.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Remaining score (Display) | `clamp(4rem, 8vw, 12rem)` | 700 | 1.0 | Dominant per-panel number — the primary at-a-glance value from 3 m |
| Player name (Heading) | `clamp(1.5rem, 3vw, 4rem)` | 600 | 1.1 | Per-panel player name above the score |
| Legs/sets won (Subheading) | `clamp(1rem, 2vw, 2.5rem)` | 600 | 1.1 | Legs/sets tally below player name |
| Stats + visit line (Body) | `clamp(0.875rem, 1.5vw, 1.75rem)` | 400 | 1.3 | Leg avg, match avg, visit line, checkout suggestion |
| Match header bar (Label) | `clamp(0.75rem, 1.2vw, 1.5rem)` | 400 | 1.2 | Game mode, out rule, format, current leg string |

**Weight rules:** Only 2 weights are used — 400 (regular) and 600/700 (bold). The remaining score uses 700 for maximum visual weight; all other emphasis uses 600.

**Letter spacing:** Remaining score uses `letter-spacing: -0.02em` to tighten the large display number. No other elements use non-default letter spacing.

### Spectator Chooser Menu (in scoring view)

Re-uses Phase 1 scoring view typography — no new sizes introduced.

| Role | Size | Weight | Usage |
|------|------|--------|-------|
| Menu item labels | 16px (Body) | 400 | "Zweites Fenster öffnen", "Anzeige hier im Vollbild" |
| Menu heading | 20px (Heading) | 600 | "Anzeigemodus" |

**Source:** CONTEXT.md Claude's Discretion + RESEARCH.md Pattern 5 (clamp values). Phase 1 sizes re-used for chooser menu.

## Color

Dark mode only. Phase 2 extends the Phase 1 token set with two display-specific surface variants.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#111318` | Page background, display root background, idle screen background |
| Secondary (30%) | `#1e2027` | Inactive player panel background, match header bar background, chooser menu background |
| Active panel surface | `#22242d` | Active player panel background — slightly brighter than secondary, distinguishable from 3 m |
| Accent (10%) | `#e8a020` | Reserved — see accent list below |
| Destructive | `#c0392b` | BUST flash background (panel-level, 2 s) |

**Accent `#e8a020` is reserved exclusively for:**
1. Active player panel top border (3px solid accent line)
2. Active player panel glow: `box-shadow: inset 0 0 40px rgba(232, 160, 32, 0.08)`
3. Checkout suggestion route text in the active player panel (e.g. "T20 T20 Bull")
4. Monitor/cast icon stroke color in the scoring view toolbar
5. Leg-win banner: player name rendered in accent color ("Leg für **ALEX**!")
6. Match-win display: winner name rendered in accent color

**Active vs. inactive panel differentiation:**
- Active player: `background: #22242d`, `opacity: 1`, `border-top: 3px solid #e8a020`
- Inactive players: `background: #1e2027`, `opacity: 0.55`, `border-top: 3px solid transparent`
- Transition: `background 200ms ease, border-color 200ms ease, opacity 200ms ease`

**BUST state:**
- Active player panel background overlaid with `rgba(192, 57, 43, 0.18)` for 2 s
- "BUST" label text: `#c0392b` (destructive), weight 700, display-size font

**Contrast verification (unchanged from Phase 1):**
- `#f0f0f0` on `#111318`: 16.2:1 — exceeds WCAG AAA
- `#f0f0f0` on `#1e2027`: 14.8:1 — exceeds WCAG AAA
- `#f0f0f0` on `#22242d`: ~14.0:1 — exceeds WCAG AAA
- `#e8a020` on `#1e2027`: 8.4:1 — passes AA and AAA
- `#c0392b` on `#111318`: 5.2:1 — passes AA

**Source:** Phase 1 UI-SPEC color tokens (pre-populated). Active panel surface `#22242d` is new in Phase 2 — derived from RESEARCH.md Pattern 4 (code example).

## Layout Contracts

### View: Spectator Display (`/display`)

Full-screen layout. No scroll. Three layers stacked via `position: fixed / absolute` and CSS `z-index`.

**Layer 1 — Base layout (always present when match active):**

```
┌────────────────────────────────────────────────────┐
│  Match Header Bar (40px fixed)                     │  z-index: 1
├──────────────┬──────────────┬──────────────────────┤
│  Player 1    │  Player 2    │  Player 3  │  P4  ... │  z-index: 0
│  (column)    │  (column)    │  (column)  │          │
│              │              │            │          │
│  [fills ~95vh − 40px]       │            │          │
└──────────────┴──────────────┴────────────┴──────────┘
```

CSS grid:
```css
.panels-grid {
  display: grid;
  grid-template-columns: repeat(var(--player-count), 1fr);
  height: calc(100dvh - 40px);
  gap: 2px;
}
/* --player-count is set via style attribute: style="--player-count:{n}" */
```

**Layer 2 — Overlays (conditional):**
- Leg/set win banner: `position: fixed; inset: 0; z-index: 10` — full-screen, semi-transparent dark overlay with centered text
- Match win display: `position: fixed; inset: 0; z-index: 20` — full-screen, opaque dark screen
- BUST flash: scoped within the player panel (`position: absolute; inset: 0`) — not full-screen

**Layer 3 — Exit/fullscreen controls (tablet only):**
- "Zurück zur Eingabe" button: `position: fixed; bottom: 24px; right: 24px; z-index: 30`
- Appears on tap anywhere on the display, auto-hides after 3 s
- Fullscreen toggle (PC): `position: fixed; top: 8px; right: 8px; z-index: 30` — small icon button, always visible

**Per-player panel layout (vertical stack inside each grid column):**

```
┌─────────────────────────────┐
│  Player Name   (heading)    │  padding-top: 24px
│  Legs: 2 / Sets: 1          │  (subheading, below name)
│                             │
│  [big remaining score]      │  flex: 1 1 auto; align-self: center
│                             │
│  Checkout: T20 T20 Bull     │  (accent, only when in finish range)
│  Visit: T20 · – · –         │  (live mid-visit OR last completed)
│  Ø Leg 52.3 · Ø Match 48.7  │  (stats line)
│                             │  padding-bottom: 24px
└─────────────────────────────┘
```

The remaining score is vertically centered in the available space between the name block (top) and the stats block (bottom). Use `display: flex; flex-direction: column; justify-content: space-between` on the panel.

**Idle state (no active match):**

```
┌────────────────────────────────────────────────────┐
│                                                    │
│              Neverman Darts                        │
│           Warte auf Match…                         │
│                                                    │
└────────────────────────────────────────────────────┘
```
Centered content, heading size for app name, body size for status text. Dark background `#111318`.

### Spectator Chooser (in scoring view)

A small floating menu triggered by the monitor/cast icon in the match view toolbar.

- Icon button: 44×44px, positioned in the existing undo-bar / toolbar row of the scoring view
- Menu appears as a bottom sheet (mobile) or inline popover (landscape) with two action buttons
- Each action button: 48px height, full width of the menu, `#1e2027` background, `#f0f0f0` label, left-aligned icon + text
- Menu heading "Anzeigemodus" 20px / 600 at top of sheet
- Dismiss on outside tap or Escape key

## Interaction Contracts

| Interaction | Behavior |
|---|---|
| Monitor icon tap (scoring view) | Opens SpectatorChooser bottom sheet / popover with two options |
| "Zweites Fenster öffnen" tap | Calls `window.open(\`${base}/display\`, '_blank', 'noopener,noreferrer')`; if null (popup blocked) shows inline text "Bitte Popups für diese Seite erlauben" |
| "Anzeige hier im Vollbild" tap | Navigates to `/display` via `goto(\`${base}/display\`)`; display page renders a "Vollbild aktivieren" prompt button on first load |
| "Vollbild aktivieren" tap (display page) | Calls `document.documentElement.requestFullscreen()` synchronously from click handler |
| Tap anywhere on display (tablet fullscreen) | Shows "Zurück zur Eingabe" button, auto-hides after 3 s; does not pause the match |
| "Zurück zur Eingabe" tap | Calls `document.exitFullscreen()` then `goto(\`${base}/match\`)` |
| Fullscreen toggle icon tap (PC window) | Toggles `requestFullscreen()` / `exitFullscreen()` on `document.documentElement` |
| Live dart entry (remote — from scoring view) | BroadcastChannel delivers `MatchState` snapshot; display re-renders affected panel immediately |
| Undo (remote) | `MatchState` snapshot delivered; display re-renders with reverted score — no special animation |
| Display opened mid-match | On mount: read `localStorage.getItem('neverman-match-snapshot')`, parse, hydrate `displayStore.state`; then subscribe to live channel |
| Display opened before any match | `localStorage` returns null; show idle screen ("Warte auf Match…"); auto-switch when first `MatchState` arrives on channel |
| BUST flash | Active panel overlaid with destructive tint; "BUST" label shown; reverts to normal after 2 s (CSS animation, no JS timer) |
| Leg win | Full-screen banner appears over the grid; holds until `currentVisit.length > 0` (first dart of next leg event-driven dismiss) |
| Match win | Full-screen match-win display replaces the grid; persists until new match starts or display is closed |

## Animation & Motion

| Element | Animation | Duration | Easing |
|---|---|---|---|
| BUST flash appear | Panel overlay fades in to `rgba(192,57,43,0.18)` | 150ms | ease-out |
| BUST flash disappear | Panel overlay fades out | 600ms | ease-in (slow fade, not abrupt) |
| BUST label appear | Scale from 0.8 + fade in | 200ms | ease-out |
| Active player switch | `opacity` and `border-color` transition on all panels | 200ms | ease |
| Leg win banner appear | Fade in + scale from 0.95 | 250ms | ease-out |
| Leg win banner dismiss | Fade out | 200ms | ease-in |
| Match win display appear | Fade in | 300ms | ease-out (matches Phase 1 MatchWinOverlay) |
| Remaining score update | No animation — instant update. Smooth enough at TV frame rates. | — | — |
| "Zurück zur Eingabe" button appear | Fade in | 150ms | ease-out |
| "Zurück zur Eingabe" button auto-hide | Fade out after 3 s idle | 300ms | ease-in |
| SpectatorChooser menu open | Slide up from bottom (mobile) or fade in (landscape) | 200ms | ease-out |

**Principle:** Animations are sub-400ms except the BUST fade-out (600ms intentional — the longer exit emphasizes the revert). No looping animations. No animations that block interaction. Remaining score updates are instantaneous — animation would obscure the live countdown effect.

## Accessibility Contracts

| Requirement | Implementation |
|---|---|
| Touch targets | SpectatorChooser icon: 44×44px minimum. Chooser menu action buttons: 48px height minimum. "Zurück zur Eingabe": 48×120px minimum. |
| Contrast | All text on display surfaces meets WCAG AA minimum 4.5:1 (all verified above — all exceed 8:1). BUST destructive tint is decorative; the "BUST" label text meets contrast independently. |
| Screen reader (display) | Display is a read-only dashboard — not the primary interaction surface. Use `role="status" aria-live="polite"` on the remaining score so score changes are announced if a screen reader is present. |
| Keyboard (chooser menu) | Tab + Enter navigable. Escape dismisses the menu. Focus returns to the monitor icon on dismiss. |
| Fullscreen API denial | If `requestFullscreen()` is denied (device/permission), display works correctly in normal browser window. No crash, no empty UI. |
| No `{@html}` | Player names and all state-derived strings rendered via `{interpolation}` only — Svelte auto-escape, consistent with Phase 1 security pattern T-03-04. |

## Sync Protocol Specification

Locked approach from CLAUDE.md + CONTEXT.md. Detailed here for executor reference.

| Property | Value |
|----------|-------|
| Channel name | `neverman-match` |
| localStorage key | `neverman-match-snapshot` |
| Message payload | Full `MatchState` JSON snapshot (not a delta) |
| Publisher | `MatchStore.dispatch()` — fires after every `reduce()` call |
| Subscriber | `DisplayStore.connect()` — called from `$effect` in `/display/+page.svelte` |
| Hydration on open | `localStorage.getItem('neverman-match-snapshot')` synchronously on `DisplayStore` instantiation |
| Tablet path | No BroadcastChannel — `/display` route reads `matchStore` directly (same in-app singleton) |
| Error handling | Both `BroadcastChannel.postMessage()` and `localStorage.setItem()` wrapped in `try/catch`; failures are non-fatal and silent (live match play is not interrupted) |

**Average computation scope for Phase 2:**
- Leg average: `player.visits` from index `legStartVisitIndex[player.id]` onward (requires `legStartVisitIndex: Record<string, number>` added to `MatchState` in reducer)
- Match average: all of `player.visits` (cross-leg, since the reducer does not reset visits at leg start — verified in reducer.ts lines 309–325)
- Format: `(totalScored / dartsThrown) * 3` — standard 3-dart average; bust visits count as 3 darts thrown, 0 scored
