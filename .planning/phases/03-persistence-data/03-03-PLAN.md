<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Imported file → local database | An untrusted `.json` file chosen by the user is parsed and used to replace all local profiles + history (replace-all, D-11) |
| Imported strings → DOM | Player names from an imported backup are later rendered (in the history list / detail via Plan 02) |
| Storage quota → app | Export/import and ongoing writes operate against a finite IndexedDB quota |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-07 | Tampering | `importAllData` consuming an untrusted backup file | mitigate | `validateImportFile()` runs `peakImportFile()` + `databaseName === 'NevermanDarts'` check BEFORE any DB write; a foreign/corrupt file is rejected with a German error and changes nothing. `importInto` runs inside a single Dexie transaction so a mid-import failure does not leave partial corruption (RESEARCH V5 / Tampering mitigations) |
| T-03-08 | Tampering | Prototype-pollution / unexpected keys when deserializing the backup | mitigate | Import uses the official `dexie-export-import` addon writing into typed Dexie tables (no assignment of arbitrary keys onto app prototypes); the databaseName structural gate rejects non-Neverman payloads. No `eval`/`Function`/`{@html}` on imported data |
| T-03-09 | Tampering | Imported player names rendered into the DOM | mitigate | All rendering (here and in Plan 02) via Svelte `{interpolation}` (auto-escaped); no `{@html}` |
| T-03-10 | Denial of Service | Storage quota exhaustion / eviction | mitigate | `requestPersistentStorage()` requested at app load (Plan 01); `getStorageWarning()` shows an >80% German warning on the Daten screen; all storage ops in try/catch — a full quota degrades gracefully and never corrupts existing data |
| T-03-SC | Tampering | npm install of `dexie-export-import` | mitigate | RESEARCH Package Legitimacy Audit verdict APPROVED (official Dexie org); Task 1 is a blocking-human verification gate on npmjs.com before install (never auto-approved) |
</threat_model>
