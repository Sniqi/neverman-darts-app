# Phase 3 — UI Design Contract

> Visual and interaction contract for Phase 3: Persistence & Data.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (SvelteKit — shadcn is React-only) |
| Preset | not applicable |
| Component library | none — hand-built Svelte components (identical to Phases 1 & 2) |
| Icon library | none — inline SVG icons where needed (consistent with Phase 2 pattern) |
| Font | system-ui stack: `system-ui, -apple-system, sans-serif` (unchanged) |

**Source:** Phase 1 + Phase 2 UI-SPECs. No new design system tooling is introduced in Phase 3. All design tokens are pre-populated from `src/app.css` (verified in codebase, 2026-06-12).

**Mandate:** Phase 3 introduces list/menu/dialog UI onto the existing app. It does NOT invent a parallel design language. Every token value below is inherited from Phases 1 & 2 unless explicitly annotated as new.

## Spacing Scale

Identical to Phases 1 & 2. Phase 3 adds no new spacing tokens.

| Token | Value | Usage in Phase 3 |
|-------|-------|------------------|
| xs | 4px | Inline gaps: separator dots between date/result/winner in history rows; label-to-value gaps |
| sm | 8px | History row internal vertical padding; list item inner gaps |
| md | 16px | Card/screen gutter (left/right padding on all Phase 3 screens); form field gaps on Daten screen |
| lg | 24px | Section spacing on the start screen between menu items; section headings on Daten screen |
| xl | 32px | Vertical margin between major sections (e.g. start screen logo area to menu area) |
| 2xl | 48px | Top/bottom padding on the start screen to vertically center menu on short phones |
| 3xl | 64px | Not used in Phase 3 |

**Exceptions:**

| Element | Value | Reason |
|---------|-------|--------|
| Start screen menu button height | 56px | Touch-first primary navigation item — larger than standard button to be reliably tappable without looking |
| History list row height | 64px minimum | Three lines of content (main line + subtitle + breathing room) at touch target scale |
| Dialog action buttons | 52px height | Destructive confirmations and resume prompt require deliberate, mis-tap-resistant targets |
| Export / Import buttons | 52px height | Match dialog action button height for visual consistency |

**Source:** Phase 1 exceptions section (touch target minimums); new exceptions derived from CONTEXT.md D-07 menu design + list row content density.

## Typography

No new sizes or weights are introduced. Phase 3 uses the exact 4-size, 2-weight scale from Phase 1.

| Role | Size | Weight | Line Height | Usage in Phase 3 |
|------|------|--------|-------------|------------------|
| Body | 16px | 400 | 1.5 | History row primary line (date · winner · result); Daten screen descriptions; dialog body text; start screen menu subtitles if any |
| Label | 14px | 400 | 1.4 | History row format subtitle ("501 Double Out"); match detail stats labels ("Ø Match"); dialog secondary text; storage warning text |
| Heading | 20px | 600 | 1.2 | Screen titles ("Match-Verlauf", "Daten / Backup"); dialog headings; section headings in detail view |
| Display | 48px | 600 | 1.0 | Not used in Phase 3 screens. Carried forward for scoring view continuity. |

**Notes:**
- History list winner name: rendered at Body (16px) size with weight **600** (not a new weight — 600 is already declared; this is the emphasis variant within the 2-weight system). Other player names in the same row use weight 400.
- Match detail view player averages: Label (14px / 400) for the label, Body (16px / 400) for the value. Consistent with Phase 2 stats line pattern.
- Start screen app title "Neverman Darts": Heading (20px / 600). Not a new Display-scale size — the start screen is a navigation hub, not a score display.
- Do NOT introduce a 5th font size (e.g. 12px for timestamps). If a timestamp needs to be smaller, use Label (14px / 400).

**Source:** Phase 1 UI-SPEC typography table (pre-populated, no changes). Phase 2 Body-size stats line pattern re-applied to detail view.

## Color

Dark mode only. Phase 3 introduces no new color tokens. All values carry forward from Phases 1 & 2.

| Role | Value | Usage in Phase 3 |
|------|-------|------------------|
| Dominant (60%) | `#111318` | Page background for all Phase 3 routes (start screen, history list, detail, Daten screen, dialogs backdrop) |
| Secondary (30%) | `#1e2027` | History list row background; dialog/bottom-sheet surface; menu card background; per-player stat blocks in detail view; Daten screen card background |
| Active panel surface | `#22242d` | Winner row highlight background in history list (replaces Secondary for the winning player's card in detail view, consistent with Phase 2 active-player treatment) |
| Accent (10%) | `#e8a020` | Reserved — see below |
| Destructive | `#c0392b` | Destructive action CTAs: "Verwerfen" button text/border; delete-match confirmation CTA; import replace-all CTA |

**Accent `#e8a020` is reserved exclusively for:**
1. All uses inherited from Phases 1 & 2 (active player border, checkout suggestion, inner bull, confirm button, undo icon, etc.)
2. **New in Phase 3:** Winner name text in the history list row (the player who won, highlighted in accent)
3. **New in Phase 3:** Winner name in the match detail view header/scoreboard
4. **New in Phase 3:** "Fortsetzen" (resume) primary CTA button background on the resume prompt

**Accent is NOT used on:** history row backgrounds, general navigation buttons, the "Daten/Backup" menu item, date/time text, stats labels, or any decorative elements.

**Destructive element styling:**
- "Verwerfen" button: outlined style — `border: 1px solid #c0392b`, text `#c0392b`, background transparent. This is a secondary action (paired with primary "Fortsetzen" which has accent background); using outlined destructive conveys danger without visual dominance.
- Delete-match CTA: filled destructive — `background: #c0392b`, text `#f0f0f0`. This is the only action in a confirmation dialog and warrants the filled treatment.
- Import replace-all CTA: filled destructive — same treatment as delete-match CTA.

**Contrast (unchanged from Phase 1 — all Phase 3 surfaces covered):**
- `#f0f0f0` on `#111318`: 16.2:1 (Dominant)
- `#f0f0f0` on `#1e2027`: 14.8:1 (Secondary)
- `#f0f0f0` on `#22242d`: ~14.0:1 (Active surface)
- `#e8a020` on `#1e2027`: 8.4:1 (Accent on Secondary — all pass AA and AAA)
- `#c0392b` on `#111318`: 5.2:1 (Destructive text — passes AA)
- `#f0f0f0` on `#c0392b`: 4.6:1 (White on filled destructive button — passes AA)

**Source:** Phase 1 + Phase 2 color tokens (pre-populated). New accent reservations derived from CONTEXT.md D-04 ("winner highlighted") and D-01 ("Fortsetzen" is the primary recovery action).

## Layout Contracts

Phase 3 introduces five new UI surfaces. All are single-column scrollable screens (max-width 480px, centered) except where stated — consistent with the Phase 1 setup screen layout pattern.

### Surface 1: Start Screen (`/`)

Single scrollable column. Max width 480px, centered. Replaces the current bare `goto('/setup')` redirect.

**Sections (top to bottom):**

```
┌────────────────────────────────┐
│  [app title: Neverman Darts]   │  20px / 600, top padding 2xl (48px)
│                                │
│  [resume prompt — if present]  │  conditional bottom sheet (see below)
│                                │
│  ┌──────────────────────────┐  │
│  │  Neues Spiel             │  │  56px height, accent background CTA
│  └──────────────────────────┘  │
│  ┌──────────────────────────┐  │
│  │  Match-Verlauf           │  │  56px height, secondary background
│  └──────────────────────────┘  │
│  ┌──────────────────────────┐  │
│  │  Daten / Backup          │  │  56px height, secondary background
│  └──────────────────────────┘  │
│                                │  gap between buttons: lg (24px)
└────────────────────────────────┘
```

**Menu button anatomy:**
- Width: 100% of column
- Height: 56px
- Padding: md (16px) horizontal
- Text: 16px / 400, left-aligned with a right-pointing inline SVG chevron (›) at trailing edge
- "Neues Spiel" button: accent background `#e8a020`, text `#111318` (dark text on accent — high contrast)
- "Match-Verlauf" + "Daten / Backup" buttons: secondary background `#1e2027`, text `#f0f0f0`
- Border radius: 8px on all menu buttons (consistent with existing card/button radii in the codebase)

**Resume prompt (inline bottom section — only rendered when an unfinished match exists in localStorage):**

Appears above the main menu as a dismissible info card, NOT as a blocking modal:

```
┌────────────────────────────────┐
│  Laufendes Spiel fortsetzen?   │  16px / 600 (heading weight within card)
│  [match info: e.g. 501 · Alex vs. Ben · 2:1 Legs]  │  14px / 400 label
│  [Fortsetzen]  [Verwerfen]     │  two buttons, side by side
└────────────────────────────────┘
```

- Card background: Secondary `#1e2027`, border-radius 8px, padding md (16px)
- Card positioned above the menu buttons with gap lg (24px) between card and first menu button
- "Fortsetzen" button: accent background, 52px height, half width minus sm/2 gap
- "Verwerfen" button: outlined destructive style (see Color section), 52px height, half width minus sm/2 gap
- Card does NOT auto-dismiss — user must explicitly tap Fortsetzen or Verwerfen

**No other modals, drawers, or popups on the start screen at initial load.** The new-match warning (D-02) is separate — it appears on the `/setup` flow entry, not on the start screen itself.

## Interaction Contracts

| Interaction | Behavior |
|---|---|
| Start screen load — unfinished match exists | Resume prompt card renders above the menu. "Neues Spiel" button behavior changes to trigger the new-match warning dialog before navigating to setup. |
| Start screen load — no unfinished match | No resume prompt rendered. "Neues Spiel" navigates directly to `/setup`. |
| "Fortsetzen" tap | Assigns saved MatchState to `matchStore.state`, navigates to `/match`. Prompt card unmounts. |
| "Verwerfen" tap (resume prompt) | Clears `localStorage['neverman-match-snapshot']`, hides prompt card. "Neues Spiel" now navigates directly to `/setup`. No navigation change unless user taps "Neues Spiel". |
| "Neues Spiel" tap — unfinished match exists and not yet discarded | Opens new-match warning dialog (Surface 2). Does NOT navigate. |
| "Verwerfen und neu starten" tap (new-match warning) | Clears snapshot, dismisses dialog, navigates to `/setup`. |
| "Abbrechen" tap (any dialog) | Dismisses dialog, returns to previous state. No data changes. |
| Tap outside a blocking dialog | No-op on new-match warning and import confirmation (must make explicit choice). Equivalent to "Abbrechen" on delete-match confirmation. |
| "Match-Verlauf" tap (start screen) | Navigates to `/history`. |
| "Daten / Backup" tap (start screen) | Navigates to `/data`. |
| History row tap | Navigates to `/history/[id]`. Full row is the tap target. |
| "Spiel löschen" tap (detail view) | Opens delete-match confirmation dialog. Does NOT delete immediately. |
| "Löschen" tap (delete confirmation) | Calls `deleteMatch(id)`, navigates back to `/history`. Deletion is immediate and permanent — no undo affordance. |
| "Exportieren" tap | Calls `exportAllData()`, which downloads `neverman-backup-YYYY-MM-DD.json` via Blob URL. Button shows loading state (disabled + spinner or label change to "Exportiere…") during async operation. Reverts to normal on completion. |
| "Datei auswählen" tap | Triggers hidden `<input type="file" accept=".json">` click. No visible state change until file is selected. |
| File selected — valid Neverman file | Immediately opens import replace-all confirmation dialog (Surface 7). No import yet. |
| File selected — invalid / non-Neverman file | Shows inline German error below the "Datei auswählen" button. No dialog opened. |
| "Importieren" tap (import confirmation) | Calls `importAllData(blob)`. Button shows "Importiere…" loading state (disabled). On success: shows inline success message "Import abgeschlossen." + updates UI. On failure: shows inline error "Import fehlgeschlagen." |
| Back chevron tap (any Phase 3 screen) | Navigates to the previous route via `history.back()` or `goto(\`${base}/\`)` for history/data screens. |

## Accessibility Contracts

| Requirement | Implementation |
|---|---|
| Touch targets | All interactive elements minimum 44×44px. Start screen menu buttons 56px height (exceeds minimum). Dialog CTAs 52px height. History rows 64px minimum height. |
| Contrast | All Phase 3 surfaces use inherited tokens — contrast ratios verified in Phase 1 (all exceed WCAG AA; most exceed AAA). No new surfaces introduced below AA threshold. |
| Dialog keyboard | Tab + Enter navigable. Escape dismisses dialogs where backdrop-dismiss is allowed. For no-backdrop-dismiss dialogs (import confirmation), Escape is equivalent to "Abbrechen". |
