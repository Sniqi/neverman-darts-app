<objective>
Build the pure statistics computation core for Phase 4: extend the engine state with two additive fields (`PlayerState.legCompleted`, `Visit.wasCheckout`), populate them in the reducer's leg-close path, and add all pure stat functions to `averages.ts`. This is the testable foundation every downstream slice (live drawer, lifetime dashboard, record detection, match-detail breakdown) consumes.

This is the MVP slice's engine layer. It is deliberately separated from UI because the stat functions must be exhaustively unit-tested before any consumer is built (RESEARCH Architectural Responsibility Map: "Pure functions over MatchState; must be unit-tested before UI"). It resolves the cross-leg average limitation flagged in `averages.ts` lines 64-70 and `PlayerStatRow.svelte` line 37.

Covers STAT-01 (cross-leg + live leg average), STAT-02 (first-9), STAT-03 (checkout %), STAT-04 (score bands), STAT-05 (highest visit / highest checkout / best+worst leg derivation primitives).

Purpose: Establish correct, tested stat primitives so all UI consumers render real values, not approximations or "‚Äî".
Output: Extended `types.ts`, `reducer.ts`, `averages.ts`; extended `averages.test.ts` and `reducer.test.ts`.
</objective>
