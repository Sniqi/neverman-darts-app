## Spacing Scale

Declared values (all are multiples of 4). Tokens defined in `src/app.css` `:root` and used via `var(--space-*)` throughout the codebase.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, stat label/value pairs, row micro-gaps |
| sm | 8px | Compact element spacing, list row internal gaps |
| md | 16px | Default card padding, content area padding, section internal gaps |
| lg | 24px | Section padding, between card groups |
| xl | 32px | Layout gaps, overlay content padding, major section breaks |
| 2xl | 48px | Page-level top padding (start screen convention) |

Exceptions:
- Touch targets (back-btn, menu buttons, drawer toggle): minimum 44×44px hit area — enforced via explicit `height` or padding, not the spacing scale. This matches the existing back-btn pattern (`width: 44px; height: 44px`).
- Stat drawer toggle button on the match view: minimum 44px height.
- Record overlay auto-dismiss duration: 2.5 s (matches the correction window timing from Phase 1). Not a spacing value but a timing constant specified here for executor clarity.

## Typography

All sizes are in px. The project uses `system-ui` — no custom web font load. Format numbers to one decimal consistently (e.g. `42.3`, not `42.30` or `42`). Source: `src/app.css`, component survey.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px | 400 | 1.5 | Screen body text, dialog body, stat values in lists |
| Label | 14px | 400 | 1.4 | Secondary metadata, dates, muted stat labels (e.g. "Ø Match:", score-band legend), error/success inline text |
| Heading | 20px | 600 | 1.2 | Screen titles, section headings (`h1`/`h2`), player name in stat rows |
| Display | 48px (fixed) / `clamp(2.5rem, 6vw, 8rem)` (responsive) | 600 | 1.0–1.1 | Record overlay main text (e.g. "180!"), win banner headline (matches existing pattern) |

Notes:
- Two weights only: 400 (regular) and 600 (semibold). No 500, no 700.
- The clamp display size is already established by `LegWinBanner` and `MatchWinDisplay` — reuse exactly.
- Chart axis labels and tick values: 14px weight 400 (same as Label role).
- Stat numbers on the lifetime dashboard: 20px weight 600 for KPI values; 14px weight 400 for their labels.

## Chart Contract

No chart library is present in `package.json`. Per CLAUDE.md "smallest runtime" principle and the static-PWA constraint, charts are hand-rolled SVG. This is consistent with the Dartboard SVG approach from Phase 1.

**Charts required (STAT-08):**

| Chart | Type | Spec |
|-------|------|------|
| Score distribution | Horizontal bar chart | Bars for 180s / 140+ / 100+ / 60+ / <60, each a `<rect>` scaled to max count. Label left (band name), value right. Accent bar for highest band with any count. |
| Average trend | Line chart | X = match index (oldest → newest), Y = match 3-dart average. Single `<polyline>` in accent color. Axis lines in `#444`. Tick labels 14px/400/`#888888`. Minimum 2 data points to render; 0–1 points shows empty state. |
| Darts per leg | Bar chart | One bar per leg in the selected match or lifetime bucket. Same spec as score distribution. |
| Win rate | Single stat display, NOT a chart | `wins / total_matches * 100` shown as `"67%"` in 48px/600/`#f0f0f0` with label "Gewinnrate" 14px/400/`#888888` below. No pie, no donut — keep it readable. |

SVG chart accessibility: Each chart `<svg>` gets `role="img"` and `aria-label="{chart name}: {plain-text summary}"`.

## Interaction Contracts

### Stat Drawer (Match View — D-01)

- Toggle button label: "Statistik ▾" (closed) / "Statistik ▴" (open). Button height: 44px minimum. Position: below ScorePanel, above the Dartboard/Numpad area.
- Drawer opens/closes with a CSS `max-height` transition (200ms ease) — no JS animation library.
- Drawer content layout: two columns of `StatCard` tiles. Left column: leg stats; right column: match stats.
- Drawer does NOT block dart input — it sits above the panel, the dartboard remains fully interactive when the drawer is open.
- Drawer state persists for the lifetime of the match route (Svelte `$state` local to the route component) — resets to closed on navigation.
- The drawer is ONLY on the input view (`/match`). The spectator view (`/display`) does NOT get a drawer (D-02).

### Record Overlay

- Appears on: `/match` view and `/display` view simultaneously via BroadcastChannel.
- Auto-dismisses after 2.5 s. No tap required, no tap to dismiss.
- Multiple records in one moment → single combined card (D-07): headline lists all records separated by " · ".
- When coincident with a leg/match win (D-08): overlay does NOT render. Instead, a `recordBadge` string is passed to the win banner/overlay component. Record text style: 16px weight 400 color `#e8a020`, below the banner subtitle.
- 180 always shows the overlay (D-04), even if not a new personal record. Text: "180!". If it IS also a new personal record (e.g. first 180): "180! · Neuer Rekord: Höchste Aufnahme".
- Animation: `fadeIn 250ms ease-out` + `scale(0.95 → 1.0)` — matches LegWinBanner exactly.
- z-index: 50 (above display elements z-10..z-20, below MatchWinOverlay z-100).

### Lifetime Stats Screen (`/stats`)

- Accessible from start screen via "Statistik" button (fourth menu item).
- Entry screen: profile picker list. Each profile is a button row (56px height, `#1e2027` background, player name 16px/400, chevron right).
- Guest players have no profile entry — they cannot appear here (D-11).
- After profile selection: ProfileStatDashboard for that profile. Back button returns to profile picker.
- If no profiles exist: empty state (see Copywriting below).
- If a profile has no completed matches: per-profile empty state (see Copywriting below).

### Match Detail Phase 4 Region

- `MatchStatBreakdown` fills the `.phase4-region` `<div>` in `history/[id]/+page.svelte`.
- Section heading: "Statistiken" (20px/600).
- Shows: per-player score bands (180s, 140+, 100+, 60+), checkout %, highest visit, highest checkout, best leg (darts).
- Layout: one `StatCard` grid per player. Player name as sub-heading (16px/600/accent for winner, 16px/600/`#f0f0f0` for others).

## Accessibility Contract

Matches the convention established in existing components:

- All player names in stat displays: Svelte `{interpolation}` only — no `{@html}` (T-03-04/T-03-05 security convention).
- Record overlay: `role="status"` + `aria-live="assertive"` (matches LegWinBanner pattern — the record is an important live update).
- Stat drawer: `aria-expanded={open}` on the toggle button. Drawer panel: `id="stat-drawer"`, toggle button gets `aria-controls="stat-drawer"`.
- Charts: each `<svg>` gets `role="img"` + `aria-label` with a plain-text summary of the data.
- Profile picker buttons: `aria-label="Statistik für {name} anzeigen"`.
- Empty states: `role="status"` on the container paragraph.
- Error states: `role="alert"` on inline error paragraphs (matches existing `.inline-error` pattern).
