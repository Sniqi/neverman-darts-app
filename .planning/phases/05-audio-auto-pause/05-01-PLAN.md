<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| localStorage → app | Audio prefs read from localStorage are attacker-influenceable only by a same-origin actor who already controls the device; coerce before use |
| game state → speechSynthesis.text | Only computed numeric scores and getSuggestion() output reach the utterance — no user free-text |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | audio-caller `utterance.text` | mitigate | Pass only the numeric visit score and `getSuggestion()` (string[]) output; never interpolate user-supplied strings; no `{@html}`/innerHTML path exists (RESEARCH Security Domain) |
| T-05-02 | Tampering | audio-prefs localStorage reads | mitigate | Coerce every read (`=== 'true'`, `Number()||default`, `?? 'de'`); fall back to typed DEFAULTS on any failure; never eval/innerHTML the values |
| T-05-03 | DoS (self) | rapid speak() queueing | accept | `speechSynthesis.cancel()` before each speak prevents backlog; worst case is a dropped utterance — no scoring impact |
| T-05-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages this phase (RESEARCH Package Legitimacy Audit: none) — no install task, no checkpoint required |

Risk level: LOW. Phase 5 adds no network, auth, or new persisted secrets; audio is purely computed output.
</threat_model>
