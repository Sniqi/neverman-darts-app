countdown lives only in the scoring window; the spectator follows via the existing `neverman-match`
BroadcastChannel using a type-discriminated `pause-tick` message (RESEARCH Pitfall 3 / D-09).

reusing the Plan 01 prefs for thresholds and the MatchWinOverlay structure for the overlay.
Output: pause `$state` + control methods on MatchStore, the `pause-tick` envelope on the existing
channel, DisplayStore pause handling, the new `PauseOverlay.svelte` mounted on both routes, and the
countdown `$effect` in `/match`.
</objective>
