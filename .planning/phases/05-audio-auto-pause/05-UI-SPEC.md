| Layer | z-index | Component |
|-------|---------|-----------|
| Leg win banner | 10 | LegWinBanner.svelte |

Rationale for z-index 60: pause overlay must appear above record celebrations (50) but below the terminal match-win overlay (100). A match cannot be won during a pause, but a record could coincide with the leg that triggered the pause — show the pause overlay on top.

## New Components — Phase 5

### 1. PauseOverlay.svelte

**Location:** `src/ui/overlays/PauseOverlay.svelte`

**Purpose:** Full-screen auto-pause countdown. Shown on both `/match` and `/display` views (CONTEXT.md D-09). Synced via the existing `neverman-match` BroadcastChannel.

**Visual anatomy:**
```
┌─────────────────────────────────────┐
│  [backdrop: rgba(17,19,24,0.96)]    │
│                                     │
│     Pause                           │  ← heading: 20px/600, --text
│     (subtitle or "Nächste Leg")     │  ← body: 16px/400, --text
│                                     │
│         MM:SS                       │  ← countdown: clamp(4rem,10vw,12rem)/600, --accent
│                                     │
│     [ Weiter ]                      │  ← CTA: accent button, height 56px
│                                     │
└─────────────────────────────────────┘
```

**Behavior:**
- Entrance: `fadeIn 300ms ease-out` (matches MatchWinOverlay pattern)
- Countdown ticks each second; digits in `--accent` color
- At 0:00, auto-resumes (no action required by user)
- "Weiter" button: `background: #e8a020; color: #111318; font-size: 18px; font-weight: 600; height: 56px; min-width: 200px; border-radius: 6px`
- "Weiter" button active state: `opacity: 0.85` (matches existing button patterns)
- `role="dialog"` with `aria-modal="true"` and `aria-label="Pause"`
- `aria-live="polite"` on the countdown digit element (announces time remaining)

**On /display (spectator view):** Identical visual. "Weiter" button is hidden on the spectator view — only the input view can resume. The spectator sees the countdown and the label only.

### 2. AudioSettings (inline section in MatchSetup)

**Location:** Inlined into `src/ui/setup/MatchSetup.svelte` (new `<section>` below Format)

**Purpose:** Caller voice toggle, SFX toggle, auto-pause configuration. Follows existing section/stepper/toggle-row patterns from MatchSetup.svelte exactly.

**Visual anatomy (within setup screen):**
```
Audio & Pause
├── [toggle-row] Anrufer (caller)         [ON/OFF switch]
├──   [only when caller ON] Sprache       [DE | EN segmented control]
├── [toggle-row] Soundeffekte             [ON/OFF switch]
├── [toggle-row] Automatische Pause       [ON/OFF switch]
├──   [only when pause ON] Pause nach     [stepper: N Legs]
└──   [only when pause ON] Pausendauer    [stepper: N Minuten]
```

**Behavior:**
- Caller OFF by default (CONTEXT.md D-06)
- SFX OFF by default (CONTEXT.md D-06)
- Auto-pause ON by default, 5 legs, 8 minutes (CONTEXT.md D-08)
- Language control (DE/EN) only renders when caller is ON — identical to the sets stepper pattern in MatchSetup (conditional section)
- Pause steppers only render when auto-pause is ON — same conditional pattern

**Toggle rows:** reuse existing `.toggle-row` / `input[type='checkbox']` CSS — no new styles needed.

**Stepper rows:** reuse existing `.stepper-row` / `.stepper-btn` CSS — no new styles needed.

**Segmented control (DE/EN):** reuse existing `.seg-control` / `.seg-btn` CSS — no new styles needed.

**Section heading:** `font-size: 20px; font-weight: 600` (matches `h2` in MatchSetup).

### 3. Audio asset files

**Location:** `static/sfx/` directory (new)

**Files:**
| File | Trigger | Max size |
|------|---------|----------|
| `static/sfx/180.mp3` | Visit score = 180 | 200 KB |
| `static/sfx/highfinish.mp3` | Checkout ≥ 100 | 200 KB |
| `static/sfx/record.mp3` | New personal record (any type) | 200 KB |

Format: MP3 primary (universal browser support). Total SFX budget: ≤ 600 KB (PWA precache budget, Phase 6 constraint, CONTEXT.md D-04).

No visual component — these are pure audio files served by GitHub Pages as static assets.

## Settings Persistence

Audio and pause preferences are stored in `localStorage` (not Dexie), matching the existing lightweight-prefs pattern from CLAUDE.md and the snapshot pattern in `match.svelte.ts`.

| Key | Type | Default | Notes |
|-----|------|---------|-------|
| `nvm_caller_enabled` | boolean | `false` | Caller voice on/off |
| `nvm_caller_lang` | `"de"` \| `"en"` | `"de"` | Caller language |
| `nvm_sfx_enabled` | boolean | `false` | Sound effects on/off |
| `nvm_pause_enabled` | boolean | `true` | Auto-pause on/off |
| `nvm_pause_legs` | number | `5` | Legs between pauses |
| `nvm_pause_minutes` | number | `8` | Countdown duration in minutes |

These are read at match setup render, written on change. They are NOT part of the match state or Dexie schema.

## Interaction Contracts

### Toggle interaction (caller, SFX, auto-pause)
- Tap/click toggles immediately; setting is written to localStorage synchronously
- No confirmation required (all reversible)
- Touch target: 44px minimum height (existing pattern)

### Pause overlay — input view (`/match`)
- Appears on leg completion that crosses the configured interval
- "Weiter" button: single tap/click resumes immediately (no confirmation)
- Pressing "Weiter" at 4:32 of the countdown stops the countdown and resumes the match
- Countdown ticks every 1 second; no fractional seconds displayed
- Format: `MM:SS` (e.g. `08:00`, `04:32`, `00:05`)

### Pause overlay — spectator view (`/display`)
- Same visual as input view, but "Weiter" button is not rendered
- Countdown is visible and legible at 3 m (clamp font size, accent color)
- State is received via BroadcastChannel; no local timer — timer lives in the match store

### Caller speech
- Fires after every CONFIRM_VISIT
- If speech synthesis is unavailable or no matching voice exists: silent (no UI feedback)
- If language preference is EN but no EN voice exists: attempt DE fallback, then silent
- Never interrupts scoring — fire-and-forget (`speak()` is non-blocking)

### SFX playback
- Fires simultaneously with the Phase 4 celebration overlay for 180s and records
- High finish (checkout ≥ 100): fires at leg completion
- Play via `new Audio(src).play()` — fire-and-forget, ignore promise rejection (browser autoplay policy)
- Volume: 0.8 (default; not user-configurable in v1.0)

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Pause overlay announced to screen readers | `role="dialog"` + `aria-modal="true"` + `aria-label="Pause"` |
| Countdown value announced | `aria-live="polite"` on countdown element; announce every 60s and at ≤10s |
| Toggle state | `role="switch"` + `aria-checked` on toggle inputs (matches existing `input[type='checkbox']` pattern) |
| CTA button label | "Weiter" — descriptive, no icon-only button |
| SFX/caller: decorative audio | Does not need ARIA labeling; visual information is never audio-only |
