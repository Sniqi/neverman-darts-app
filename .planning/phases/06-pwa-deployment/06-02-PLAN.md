<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → service worker | The Workbox SW intercepts same-origin fetches within scope /neverman-darts-app/ and serves from precache |
| origin → SW scope | SW scope is bounded to the subpath; it cannot intercept cross-origin or out-of-scope requests |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-03 | Elevation of Privilege | SW registered at wrong/too-broad scope | mitigate | Explicit scope = base + '/' in SvelteKitPWA config; verified in DevTools (Task 4 step 5). Scope is same-origin only — cannot intercept other origins (RESEARCH Security Domain). |
| T-06-04 | Information Disclosure | Stale content served by an outdated SW | mitigate | registerType 'prompt' + includeVersionFile true ensure a new SW is detected and the user is prompted (PLAT-03, surfaced via ReloadPrompt). |
| T-06-05 | Information Disclosure | Sensitive data leaked via precache | accept | Precache holds only JS/CSS/HTML/PNG/SVG/MP3/webmanifest — no tokens, keys, or PII. Dexie IndexedDB (history/profiles) is never precached (RESEARCH Security Domain). |
| T-06-06 | Spoofing/Tampering | SW served over insecure transport | accept | Service workers require a secure context; GitHub Pages enforces HTTPS automatically (RESEARCH V9). Not applicable to localhost preview which is also a secure context. |
</threat_model>
