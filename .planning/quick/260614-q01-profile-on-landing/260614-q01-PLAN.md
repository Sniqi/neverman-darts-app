# Quick Task 260614-q01: Profile management on landing page + back button on /setup

## Objective

1. Add a collapsible "Spieler verwalten" section to the landing page (`+page.svelte`) using the same toggle pattern that exists in `MatchSetup.svelte`.
2. Add a "Zurück" back button to `MatchSetup.svelte` that navigates to root (`/`).

## Tasks

### Task 1 — Landing page: add ProfileManager

**File:** `src/routes/+page.svelte`

Add imports for `ProfileManager` and a `profilesOpen` toggle state.  
Add a "Spieler verwalten" toggle button to the nav (same visual style as the other `menu-btn` entries), placed below the existing menu buttons.  
When toggled open, render `<ProfileManager />` below the nav.  
Copy the `.profiles-toggle` + `.toggle-arrow` styles from `MatchSetup.svelte` into `+page.svelte` so the button matches the landing page's card style instead.

**verify:** Clicking "Spieler verwalten" toggles the ProfileManager open/closed. Profiles can be created, edited, deleted from the landing page.

### Task 2 — Setup page: add back button

**File:** `src/ui/setup/MatchSetup.svelte`

Add a back button at the top of `<main class="setup-screen">` — a small `←` chevron link that calls `goto(`${base}/`)`.  
Style: minimal, sits above the `<h1>`, no heavy styling — just an inline `button` or `<a>` with a back arrow and "Zurück" text, color `#888`.

**verify:** Navigating to /setup shows a back button at the top. Clicking it goes back to the landing page.
